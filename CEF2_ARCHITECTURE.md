
# CEF-2 Upsilon Creative Edge Engine

## Purpose
CEF-2 treats the upsilon node as a **creative-edge controller**:
- not pure harmony
- not pure chaos
- but a regulated band where novelty, identity, and resonance coexist

## Node anatomy
Each node has 7 channels:
1. Purple = source
2. Blue = memory
3. Green = growth
4. Yellow = attention
5. Orange = desire
6. Red = action
7. Silver = contrast / correction

Each node also contains:
- identity core
- memory trace
- fracture-risk accumulator
- creative-edge shell

## Metrics
The engine computes:
- coherence
- novelty
- memory continuity
- overload
- fragmentation
- source alignment
- desire tension
- risk
- creative edge score

## Biometrics bridge
This prototype uses biometrics as **modulators**, not as the mind itself.

Current input channels:
- HRV: calm / recovery capacity
- EDA: sympathetic arousal
- EMG: effort / muscular intent
- Respiration: agitation vs. regulated breathing
- IMU: motion / activity
- Temperature: thermal comfort proxy

These modulate:
- source gain
- desire gain
- shadow gain
- recovery gain
- exploration gain
- memory gain
- attention gain

## VR skin route
A practical non-invasive VR/AR path is:

### Layer 1: wellness / interaction
- PPG or ECG-derived HRV
- EDA
- respiration belt or chest expansion
- IMU in gloves / sleeves / suit
- optional localized EMG

Use case:
- adaptive haptics
- creative-state detection
- stress-aware world response
- resonance training
- player-AI-source feedback loop

### Layer 2: assistive interaction
- HD-EMG sleeve or compact EMG bands
- IMU + joint angle estimation
- pressure / tactile pads
- optional EEG if needed for research

Use case:
- gesture decoding
- exosuit intention assist
- prosthetic hand / limb shared control
- rehabilitation prototypes

### Layer 3: regulated medical branch
This is separate from the entertainment branch.
Possible use:
- rehab companion system
- adaptive training feedback
- assistive device intention decoding
- clinician-facing summaries

Do not merge directly into diagnosis or treatment claims without a formal medical-device path.

## Current prototype limits
This is intentionally early-stage.
It is useful for:
- structure exploration
- control-law experimentation
- signal mapping
- visualization

It is **not**:
- a validated physiological model
- a clinical model
- a prosthetic controller ready for human use
- a safety-tested haptics controller
