# In-Game Skin Design Notes

## Concept

The skin is a playable field layer, not just a texture.

It behaves like a living resonance membrane that can belong to:
- player
- AI agent
- animal / companion
- hybrid creature

## Core functions

1. **Receive**: read local body and world state.
2. **Translate**: convert coherence / novelty / source / desire / fracture into body response.
3. **Emit**: show state through glow, ripple, motion, haptics, and environmental resonance.
4. **Protect**: soften overload when fracture risk rises.
5. **Bond**: synchronize with nearby entities to form shared globes.

## Shared data fields

- creativeEdge
- coherence
- novelty
- identity
- fractureRisk
- sourceAlignment
- desireTension
- shadowPressure
- 7 channel amplitudes
- biometrics or body-state estimates

## Species tuning ideas

### Player skin
- clear seven-channel color expression
- controller/vest/glove vibration
- source shimmer when aligned
- fracture crackle when overloaded

### AI skin
- less literal body feedback
- stronger globe linking effects
- more visible source/desire conflict
- possibility of symbolic geometry extruding from shell

### Animal skin
- body heat pulse
- breathing-linked glow
- herd/caretaker synchronization
- ear/tail/fur/wing resonance rhythm

## Medical-adjacent extension

Keep separate from the game branch.
Use the same data architecture but isolate safety logic, actuation boundaries, and claims.
